# KAIROS

Experimental auto-trader. Runs `scan_htf` (High Tight Flag breakout) +
`compute_bbtc` (state-based trend follower) natively in Python on
superotter, opens long positions tagged HTF / BBTC / BOTH, manages
exits with strategy-specific stops, and writes state files that the
stockotter dashboard reads via the `/api/kairos/*` proxy.

## Local development

```bash
cd python/kairos
uv sync --extra test
# Run the dashboard locally
uv run uvicorn dashboard_web:app --reload --port 8082
# Run the bot locally (needs FMP_API_KEY + BOT_API_KEY + reachable stockotter)
KAIROS_MODE=paper FMP_API_KEY=... BOT_API_KEY=... uv run python -m kairos_trading.run --state-dir ./state
```

## Parity tests

The TS implementations in `server/signals/strategies/{htf,bbtc}.ts` are
the canonical spec. The parity tests in `tests/` verify Python output
matches a JSON baseline generated by the TS implementation.

```bash
# 1. Generate baselines (from the repo root)
npm run kairos:baseline
# 2. Run Python parity tests
cd python/kairos
uv run pytest tests/
```

`tests/test_synthetic_setups.py` is additive coverage that runs without
the TS baseline — useful as a smoke test in environments lacking Node.

## Deploy to superotter

```bash
# From the repo root, with the SSH key already authorized:
scp -r python/kairos administrator@10.209.32.8:/home/administrator/
ssh administrator@10.209.32.8
cd /home/administrator/kairos
cp .env.example .env && nano .env  # fill in FMP_API_KEY + BOT_API_KEY
docker compose up --build -d
docker compose logs -f kairos-bot
```

Then verify from the stockotter side: open `/kairos` in the browser —
the offline pill should flip to "online" within ~15 s, and the
watchlist should populate on the next bot tick.
